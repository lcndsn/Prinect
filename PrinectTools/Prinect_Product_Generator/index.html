1 O que é esta plataforma?

Esta plataforma gera arquivos JDF para o sistema Prinect da Heidelberg de forma simples, sem precisar conhecer o Prinect diretamente.

JDF (Job Definition Format) é o formato que o Prinect usa para receber informações de um trabalho de impressão. Você preenche um formulário, clica em um botão e o arquivo JDF é criado e baixado automaticamente.

💡 Fluxo resumido: Preencha os dados → clique em Gerar e Baixar JDF → mova o arquivo para a pasta do servidor → o Prinect importa automaticamente.

3 Cadastrar um produto

Na aba Novo Produto, preencha os campos de cima para baixo:

  1. Customer ID e Customer Name — código e nome do cliente que encomendou o trabalho.
  2. Job Name — nome do trabalho. O Article Name é preenchido automaticamente com o mesmo valor; edite-o se precisar de um nome diferente.
  3. Article ID — clique em ⚡ para gerar o próximo número automático, ou digite um código específico se o cliente já tiver um.
  4. Shape ID — código do projeto de embalagem. Clique em ⚡ para automático ou digite manualmente. Veja detalhes na seção 6.
  5. Job ID — aparece automaticamente: é o prefixo + Article ID. Não é editável.
  6. Clique em ⬇ Gerar e Baixar JDF.
💡 Nenhum campo pode ficar em branco. Se o botão não funcionar, procure o campo destacado que falta preencher.

5 Gerar o JDF e onde salvar

Com todos os campos preenchidos, clique em ⬇ Gerar e Baixar JDF. O arquivo é baixado automaticamente para a pasta Downloads do seu computador com o nome JobID_NomeProduto.jdf.

Uma janela de confirmação abrirá mostrando o nome do arquivo e o caminho de destino configurado:

Exemplo de caminho de destino
\\servidor\PTConfig\Connector\JDFImport

Mova o arquivo da pasta Downloads para esse caminho. O Prinect monitora essa pasta e importa o arquivo automaticamente.

💡 Use o botão 📋 Copiar caminho na janela de confirmação para copiar o endereço e colar direto no Explorador de Arquivos do Windows.
⚠ O caminho de destino é configurado em Configurações → Caminho de Destino do JDF. Se aparecer "(não configurado)", faça isso antes de continuar.

6 Shape ID: automático ou manual

O Shape ID identifica o projeto de embalagem. Você tem duas opções:

  • Automático (⚡): clique no botão e o sistema gera o próximo número sequencial. O contador avança a cada clique.
  • Manual: digite o código fornecido pelo seu departamento de criação (ex.: HBR_1234). O contador interno não é afetado.

O contador do Shape ID é independente do contador do Article ID e pode ser redefinido em Configurações → Contador de Shape ID.

💡 Se o departamento de criação usa numeração própria, sempre use o código dele no campo Shape ID — não clique em ⚡.

7 Histórico de produtos

A aba Histórico guarda todos os produtos já cadastrados. O que você pode fazer lá:

  • Buscar qualquer produto pelo nome, código ou cliente
  • Baixar novamente o JDF de qualquer produto já gerado (botão ⬇ na linha)
  • Ver o caminho do servidor que estava configurado no momento do cadastro
  • Excluir um registro (botão 🗑) — só remove do histórico, não apaga o JDF baixado
  • Ver os indicadores no topo: total de produtos, clientes únicos, próximo ID

8 Backup: exportar e importar

Os dados ficam salvos no navegador. Se limpar o cache ou trocar de computador, os dados somem. Por isso, faça backup regularmente:

  • Exportar JSON: salva todos os registros num arquivo .json. É o backup completo — guarde em pasta segura ou no servidor.
  • Exportar CSV: gera uma planilha que abre direto no Excel, útil para relatórios ou envio por e-mail.
  • Importar JSON: carrega um backup anterior. Os registros são adicionados aos existentes — duplicatas são ignoradas automaticamente.
⚠ Nunca limpe o histórico do navegador sem antes exportar o JSON. Dados apagados do navegador não têm recuperação.

9 O que fazer com o arquivo JDF?

Após baixar o .jdf e mover para a pasta do servidor:

  1. O Prinect Connector detecta o arquivo automaticamente na pasta JDFImport.
  2. O trabalho aparece no Prinect Cockpit como um novo job aguardando processamento.
  3. Abra o job no Cockpit e anexe o arquivo de arte (pdf) manualmente.
  4. Siga o fluxo normal de pré-impressão no Prinect.
💡 Se o arquivo JDF não aparecer no Cockpit após alguns minutos, verifique se o serviço Prinect Connector está ativo no servidor e se o caminho configurado está correto.
⚠ Para dúvidas técnicas sobre o Prinect Connector ou o Cockpit, consulte o suporte da Heidelberg ou o manual do Prinect Integration Manager.
